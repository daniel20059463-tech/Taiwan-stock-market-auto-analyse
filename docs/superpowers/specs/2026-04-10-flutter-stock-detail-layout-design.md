# Flutter 個股詳細報價頁版面設計

日期：2026-04-10

## 目標

為 Flutter 股票 App 設計一個不需頻繁上下滑動即可看到核心資訊的「個股詳細報價」頁面。頁面要符合台股看盤習慣，維持深色模式、高資訊密度、紅漲綠跌，並在單一手機畫面內完整呈現：

1. 頂部標題與即時價
2. 中段主視覺功能切換
3. 下段報價區（最佳五檔 / 分時明細）
4. 底部操作列

## 設計原則

- 使用四段式固定結構，不做長頁式堆疊。
- 優先使用垂直空間比例，但要加入最小高度保護。
- 當螢幕高度不足時，只允許中段與下段內容區各自內部捲動；整頁本身不做長捲動。
- 關鍵交易操作固定在螢幕底部。
- 頂部報價資訊與中段圖表維持視覺中心，避免被下方表格擠壓。

## 整體版面分配

頁面由上到下固定分成四個區塊：

### 1. Header 區（約 15%）

用途：
- 顯示股票名稱 / 代號
- 顯示超大成交價
- 顯示漲跌與漲跌幅
- 顯示開 / 高 / 低 / 收 / 量

內容結構：
- 第一列
  - 左：股票名稱 + 代號
  - 中：超大成交價
  - 右：漲跌 / 漲跌幅
- 第二列
  - 緊湊欄位列：開、高、低、收、量

設計要求：
- 成交價為頁面最大字級
- 漲跌顏色遵守台股標準：
  - 紅：上漲
  - 綠：下跌
  - 黃或白：平盤
- 第二列欄位採等寬或近等寬排列，避免換行

最小高度建議：
- `110px`

### 2. 功能切換區（約 30%）

用途：
- 承載個股主視覺資訊
- 用 Tab 切換三種視圖

Tab 項目：
- 走勢圖（預設）
- K線圖
- 技術指標

內容結構：
- 上方固定 `TabBar`
- 下方為對應內容視圖

設計要求：
- 第一版先保證三個 tab 的容器完整切換
- 預設頁籤為「走勢圖」
- 圖表區不被下方報價區擠壓

最小高度建議：
- `220px`

### 3. 報價區（約 45%）

用途：
- 同時顯示最佳五檔與最新逐筆成交

版面：
- 左右 `50% / 50%` 分割

左半部：最佳五檔
- 上半區：委賣 5 檔
- 下半區：委買 5 檔
- 顯示：
  - 檔位
  - 價格
  - 掛單量

右半部：分時明細
- 垂直列表顯示最新成交
- 顯示：
  - 時間
  - 價格
  - 單量
- 內盤：綠色
- 外盤：紅色

設計要求：
- 左右兩區都要有各自標題列
- 內容區可個別捲動
- 不讓其中一側內容過多時撐高整體頁面

最小高度建議：
- `260px`

### 4. Action Bar（約 10%）

用途：
- 承載快速交易與常用操作

內容：
- 左側 icon 操作
  - 加入自選
  - 警示設定
- 右側兩個大按鈕
  - 買進（紅色）
  - 賣出（綠色）

設計要求：
- 固定於最底部
- 不參與頁面捲動
- 按鈕必須醒目且可單手點擊

最小高度建議：
- `72px`

## 響應式策略

### 高度策略

採用「比例優先 + 最小高度保護」：

- Header：15%，最小 `110px`
- 功能切換區：30%，最小 `220px`
- 報價區：45%，最小 `260px`
- Action Bar：10%，最小 `72px`

若螢幕可用高度足夠，直接依比例分配。

若螢幕高度不足：
- 優先保留 Header 與 Action Bar 的可讀性與可操作性
- 功能切換區與報價區縮到最小高度後，內容改為內部捲動
- 不讓整頁變成長頁式捲動

### 寬度策略

報價區固定左右對半：
- 左：最佳五檔
- 右：分時明細

若螢幕過窄：
- 優先壓縮欄位內 padding
- 其次縮小次要字級
- 不改成上下堆疊

## Flutter 實作架構建議

建議在 `stock_detail_quote_page.dart` 使用：

- `Scaffold`
- `SafeArea`
- `LayoutBuilder`
- `Column`

區塊元件切分建議：

- `StockDetailHeaderSection`
- `StockDetailChartTabsSection`
- `StockDetailQuotePanelsSection`
- `StockDetailActionBar`

其中：
- `StockDetailQuotePanelsSection` 內部再切成
  - `OrderBookPanel`
  - `TimeAndSalesPanel`

## 版面配置建議

建議透過 `LayoutBuilder` 取得可用高度，先算出四段高度，再套用 `clamp`：

- `headerHeight`
- `chartHeight`
- `quoteHeight`
- `actionBarHeight`

避免只使用單純 `Expanded(flex: ...)`，因為在小螢幕下會讓 Header 與 Action Bar 被壓縮到不可用。

## 第一版範圍

這一版只處理版面與容器結構，不處理：

- 真實行情串接
- 真實下單
- 五檔與逐筆資料流更新
- 技術指標計算
- 圖表互動手勢優化

第一版交付標準：
- 四個區塊完整存在
- 比例與最小高度保護生效
- 中段與下段內容在小螢幕時可內部捲動
- 買進 / 賣出操作列固定於底部

## 驗收標準

### 視覺驗收

- 不需整頁上下捲動即可看到 Header、功能切換區、報價區與底部操作列
- 中段與下段比例清楚
- 最佳五檔與分時明細左右各半
- Header 的成交價、漲跌與基本欄位層級清楚

### 互動驗收

- `TabBar` 可切換三種視圖
- 下方兩個報價面板在內容超出時可獨立捲動
- 底部操作列固定可點擊

### 工程驗收

- Flutter `analyze` 無錯誤
- Widget tests 可驗證四段式佈局與底部操作列存在
