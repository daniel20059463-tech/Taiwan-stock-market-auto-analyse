# 2026-04-03 首頁改版設計

## 目標

把首頁從目前偏長、偏分散的資訊堆疊，調整成長寬適中、桌面一屏內大部分重點可讀完的版型。

新版首頁優先服務以下使用情境：

- 先快速掃描全市場機會
- 再觀察類股熱度
- 最後切到單一標的詳細盤面

整體視覺方向採用：

- 資訊清楚
- 比較輕鬆閱讀
- 不走過度壓迫的專業終端風格

## 使用者偏好確認

這次首頁設計已根據使用者明確選擇收斂為以下方向：

- 首頁主角：全市場機會總覽
- 首頁第一視角：全市場清單 + 類股熱度
- 全市場清單主排序：漲跌幅 / 成交量異動最大
- 類股熱度呈現：排行列表
- 首頁下半部保留：單一標的詳細盤面
- 預設選中標的：最近正在持有 / 剛交易過的標的
- 全市場清單型式：10 檔左右的大卡列
- 詳細盤面型式：大圖表 + 右側摘要
- 採用版型方向：A 市場優先型

## 首頁資訊架構

首頁拆成三層，而不是目前多塊區域上下堆疊的形式。

### 第一層：市場總覽條

用途：快速確認系統是否正常運作。

保留資訊：

- 連線狀態
- 追蹤總數
- 篩選結果
- 資料節流狀態
- 資料來源
- 最後更新時間

設計原則：

- 單列呈現
- 不超過一列換行
- 每張摘要卡只放一個數值重點

### 第二層：市場總覽主區

左右雙欄。

左側為 `全市場機會清單`：

- 顯示約 10 檔大卡列
- 預設排序依據：
  - 漲跌幅絕對值
  - 成交量 / 成交額異動
- 每張卡顯示：
  - 股票代碼 + 中文名稱
  - 類股中文名稱
  - 目前狀態標籤
  - 現價
  - 漲跌幅
  - 成交額或量能摘要
- 事件分數不作主欄位，只作輔助資訊

右側為 `類股熱度排行`：

- 顯示前 6 類股
- 每列顯示：
  - 類股名稱
  - 熱度分數或熱度等級
  - 類股平均漲跌幅
  - 代表股
- 採排行列表，不採熱力圖

### 第三層：單一標的詳細盤面

左右雙欄。

左側為 `大圖表區`：

- 即時 / 歷史切換
- K 線或分時圖
- MA5 / MA20 / MA60
- 標的切換控制列

右側為 `摘要與帳本區`：

- 價格結構
- 技術位階
- 量能監控
- 最新 K 棒
- 持倉摘要
- 最近成交

設計原則：

- 主圖為焦點
- 右側摘要為輔助
- 不再讓右側卡片高度過長

## 首頁預設選股規則

首頁載入時，單一標的詳細盤面的預設標的依以下優先順序決定：

1. 目前持有中的標的
2. 最近成交過的標的
3. 若以上都沒有，退回全市場清單第一名

切換規則：

- 點擊全市場卡列時，下半部詳細盤面同步切換
- 點擊類股熱度排行時，上半部全市場清單套用該類股篩選
- 若篩選後選中標的不在目前結果中，自動切換到新的第一名

## 視覺與排版原則

### 高度控制

新版首頁以桌面一屏閱讀為優先，不主動追求一頁塞入所有資訊。

原則如下：

- 市場總覽條固定為一層
- 市場總覽主區高度控制在中等
- 單一標的詳細盤面保留足夠圖表高度，但避免再往下長出多層附屬卡片
- 不在首頁保留次要區塊，例如過長的訊號明細表

### 版型比例

建議比例：

- 第二層：全市場清單 1.55 / 類股熱度 0.95
- 第三層：詳細盤面 1.5 / 摘要帳本 1

這樣可以維持市場總覽是主角，同時保留單一標的大圖表。

### 文字層級

- 頁面主標題：最高層級
- 區塊標題：第二層級
- 卡片數值：高於內文
- 輔助說明：一致降為 muted 色
- 中文文案全部統一，不保留英文混用

## 不在本次首頁改版範圍

以下內容不是本輪首頁設計的核心，不在這份規格內擴張：

- 改動後端行情資料流
- 修改策略邏輯或風控邏輯
- 增加新的交易功能
- 重做其他頁面的資訊架構
- 將首頁擴成多面板可拖拉工作區

## 元件調整方向

### AppShell

- 側邊欄維持現有路由結構
- 僅保留必要的系統狀態摘要
- 不與首頁主區競爭視覺焦點

### Dashboard

`Dashboard` 需要從目前的多區塊堆疊式布局改成：

- 單列市場總覽條
- 雙欄市場總覽主區
- 雙欄標的詳細盤面

並調整下列內容：

- 訊號列表不再作首頁主角
- 即時異動改為全市場大卡列
- 類股熱度從附屬資訊提升為首頁核心區
- 流量控制維持在可見，但降級成輔助資訊

## 狀態與資料需求

首頁只使用現有前端 store 與 worker 已提供資料，不新增新的資料契約。

需要直接使用的資料：

- `snapshot.symbols`
- `connectionState`
- `selectedSymbol`
- `portfolio.positions`
- `portfolio.recentTrades`
- `historyCache`
- `sessionCache`
- `tick delta`

類股熱度排行可先用現有快照做聚合：

- 依類股分組
- 取平均漲跌幅
- 統計活躍檔數
- 以代表股作顯示

## 錯誤與空狀態

首頁必須保留以下空狀態處理：

- 沒有 snapshot 時，用 fallback instruments 建立基本畫面
- 沒有持倉與最近成交時，預設標的退回市場第一名
- 沒有類股熱度資料時，顯示空清單而不是錯誤訊息
- 沒有 session / history cache 時，仍可先畫 fallback K 棒

## 驗收標準

### 版型

- 首頁不再呈現明顯過長的上下堆疊感
- 在桌面常見尺寸下，首頁主要資訊可在一屏內看到大部分內容
- 主視線順序為：
  - 市場總覽
  - 全市場清單
  - 類股熱度
  - 單一標的詳細盤面

### 可讀性

- 所有首頁主要文字皆為乾淨中文
- 同類資訊卡使用一致字級與間距
- 主標題、區塊標題、卡片數值、輔助說明的視覺層級明確

### 互動

- 點選全市場卡列可同步切換詳細盤面
- 點選類股熱度排行可套用清單篩選
- 首頁預設標的遵守持倉 / 最近交易優先規則

### 技術

- 不破壞既有 worker / store / chart 流程
- `npm test` 通過
- `npm run build` 通過
